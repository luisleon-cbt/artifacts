Objective 2.2 · Switching Technologies & Features

How switches keep frames moving.

Switches forward frames based on MAC addresses, but a modern enterprise switch does far more: VLANs to segment broadcast domains, 802.1Q trunks to carry many VLANs over one link, Spanning Tree to prevent loops in redundant topologies, link aggregation to combine bandwidth, port security to lock down access, and DHCP snooping to block rogue servers. This artifact walks through each — and lets you watch STP elect a root bridge and trace a frame across a VLAN topology.

The complete reference.

A switch operates at Layer 2 of the OSI model. It learns where MAC addresses live by watching the source addresses of incoming frames (building its MAC address table, also called the CAM table), and forwards frames based on the destination MAC. Beyond basic forwarding, modern switches implement a stack of features for segmentation, redundancy, and security.

VLANs — Virtual LANs
VLAN BASICS

One broadcast domain per VLAN

A VLAN logically segments a switch into separate Layer-2 networks. Devices in different VLANs cannot reach each other without a router or Layer-3 switch — even on the same physical switch.

VLAN 1: Default VLAN on most switches. Best practice: don't use it for user traffic.

VLAN range: 1-4094 (with reserved IDs at the high end). 12-bit field in the 802.1Q tag.

ACCESS PORT

One VLAN, untagged

A switch port assigned to a single VLAN. Connects to end devices (PCs, printers, IoT). Frames enter and leave untagged — the switch handles VLAN tagging internally.

CONFIG: switchport mode access + switchport access vlan 10

TRUNK PORT

Many VLANs, tagged

A port that carries traffic for multiple VLANs simultaneously. Used between switches or between a switch and a router. Frames are tagged with their VLAN ID using 802.1Q.

CONFIG: switchport mode trunk + switchport trunk allowed vlan 10,20,30

VOICE VLAN

VoIP phone on the same port as a PC

A special access-port config where a phone passes a tagged voice VLAN to the switch and the PC plugged into the phone uses the untagged data VLAN. One physical port, two VLANs.

CONFIG: switchport voice vlan 100 on top of access config

802.1Q Trunking

The IEEE standard for tagging frames with a VLAN ID. The 802.1Q tag is a 4-byte field inserted into the Ethernet header between the source MAC and the EtherType. It contains a 12-bit VLAN ID and a 3-bit priority field (used for class-of-service).

Native VLAN: One special VLAN on a trunk is sent untagged — the "native" VLAN. By default this is VLAN 1. Both ends of a trunk must agree on the native VLAN; a mismatch causes traffic to leak between VLANs and STP confusion.

Inter-VLAN Routing — three ways
ROUTER-ON-A-STICK

One router, sub-interfaces, one trunk

A single physical router interface divided into sub-interfaces (Gi0/1.10, Gi0/1.20...), each handling one VLAN via 802.1Q. The trunk between switch and router carries all VLANs.

USE: Small networks, fewer than 5-6 VLANs. Cheap.

LAYER 3 SWITCH (SVI)

Switch Virtual Interfaces

A multilayer switch creates one virtual interface per VLAN (interface vlan 10) and routes between them at wire-speed using hardware ASICs. The modern enterprise approach.

USE: Any campus network with multiple VLANs.

Spanning Tree Protocol (STP) — Loop Prevention

If two switches connect to each other with two cables (for redundancy), a broadcast frame loops forever because Ethernet has no TTL. STP solves this by logically blocking redundant paths — only opening them if the primary path fails. The protocol elects a single root bridge and computes a loop-free path from every switch to that root.

VariantStandardConvergenceNotes
STP (classic) 802.1D 30-50 sec Original (1985). One STP instance for all VLANs.
RSTP 802.1w ~6 sec Rapid STP. Most common variant today. One instance for all VLANs.
MSTP 802.1s ~6 sec Multiple instances — group VLANs into instances, each runs its own STP.
PVST+ / RPVST+ Cisco ~6 sec Per-VLAN STP — one STP instance per VLAN. Cisco proprietary.

STP Port States (classic 802.1D): Blocking → Listening → Learning → Forwarding (or Disabled). RSTP simplifies to Discarding → Learning → Forwarding.

PORTFAST

Skip the wait on access ports

On a port connected to a single end device (PC), going through the full STP convergence cycle is wasteful. PortFast jumps directly to Forwarding state. Always pair with BPDU Guard.

BPDU GUARD

Shut down ports that receive BPDUs

If a port configured for PortFast receives a BPDU (meaning someone plugged in a switch), it immediately shuts the port. Prevents accidental loops from rogue switches.

ROOT GUARD

Prevents a downstream switch from becoming root

On trunk ports facing the access layer, blocks BPDUs that would make a downstream switch take over the root bridge role.

LOOP GUARD

Protects against unidirectional link failures

If BPDUs stop arriving on a non-designated port, instead of unblocking (and potentially creating a loop), loop guard places the port in a loop-inconsistent state.

Link Aggregation — Bundling Ports

Combine multiple physical links into one logical link for higher bandwidth and redundancy. If one cable fails, the bundle continues operating with reduced bandwidth.

LACP (802.3ad)

Open standard

Link Aggregation Control Protocol. Works across vendors. Sends LACPDUs to negotiate the bundle. Modes: active (initiates negotiation) and passive (responds only). Both sides cannot be passive.

PAgP

Cisco proprietary

Port Aggregation Protocol — Cisco's version. Modes: desirable (initiates) and auto (responds only). Both sides cannot be auto. Use LACP for new deployments.

Port Security

Limits which MAC addresses can use a switch port. Detects port theft (someone unplugging a PC and plugging in their own laptop) or rogue device attachment.

  • Maximum MAC addresses — e.g., 2 per port (allows a phone + PC). Frames from any additional MAC trigger a violation.
  • Sticky MAC — port dynamically learns the first N MACs and saves them to running-config. Convenient for unmanaged endpoint inventories.
  • Violation actions:
    • Protect — silently drop frames from unknown MACs (no logging).
    • Restrict — drop frames AND log/counter the violation.
    • Shutdown (default) — disable the port, requires manual no shutdown to recover.
DHCP Snooping & Dynamic ARP Inspection (DAI)
DHCP SNOOPING

Block rogue DHCP servers

Configure switch ports as trusted (DHCP-server-facing) or untrusted (user-facing). The switch only forwards DHCP server responses (OFFER, ACK) from trusted ports. Blocks an attacker from running a fake DHCP server on a user port.

BUILDS: the DHCP snooping binding table — a list of legitimate IP-MAC-port mappings.

DYNAMIC ARP INSPECTION (DAI)

Block ARP spoofing

Uses the DHCP snooping binding table to validate ARP messages. Drops ARP replies where the IP-MAC pair doesn't match a legitimate entry. Stops man-in-the-middle attacks.

DEPENDS ON: DHCP snooping must be enabled first.

IP SOURCE GUARD

Block spoofed source IPs

Also uses the DHCP snooping binding table. Drops packets whose source IP doesn't match the binding for that port. Stops IP spoofing on access ports.

802.1X

Port-based authentication

Before a port allows any traffic, the connected device must authenticate (typically via RADIUS). Often paired with a NAC (Network Access Control) system to enforce policy based on user identity.

Jumbo Frames, Flow Control, PoE
JUMBO FRAMES

MTU greater than 1500

Standard Ethernet MTU is 1500 bytes. Jumbo frames raise it to typically 9000 bytes, reducing per-frame overhead for high-bandwidth flows. Common for storage networks (iSCSI, NFS) and backups.

CATCH: all devices in the path must agree on MTU. A mismatch causes mysterious packet drops.

FLOW CONTROL (802.3x)

Pause frames

When a switch buffer fills, it can send a PAUSE frame to the source asking it to stop transmitting briefly. Largely deprecated in favor of QoS but appears on exam questions.

POE / POE+ / POE++

Power over Ethernet

PoE (802.3af): ~15W per port — IP phones, basic APs.
PoE+ (802.3at): ~30W per port — modern APs, PTZ cameras.
PoE++ Type 3 (802.3bt): 60W — high-power APs, displays.
PoE++ Type 4 (802.3bt): 90W+ — pan/tilt cameras, kiosks.

MAC ADDRESS TABLE

The switch's brain

Also called the CAM table (Content-Addressable Memory). Lists every MAC address the switch has seen, mapped to the port it came from. Entries age out after a default ~5 minutes. show mac address-table displays it.

Common Exam Traps
  • Native VLAN must match on both ends of a trunk. A mismatch lets traffic leak between VLANs (security risk) and causes STP confusion.
  • Access port = 1 VLAN; trunk port = many VLANs (tagged). The phrase "carries multiple VLANs" always points to a trunk.
  • STP root bridge: lowest priority wins; lowest MAC is the tiebreaker. Default priority is 32768; admin sets it lower (e.g., 4096) on the switch you WANT as root.
  • RSTP convergence ~6 seconds vs classic STP ~30-50. If a question says "fast convergence required" → RSTP.
  • LACP = open standard. PAgP = Cisco only. Multi-vendor scenario → LACP.
  • DAI requires DHCP snooping to work. They're paired — DAI can't validate anything without the snooping binding table.
  • Port security violation default = shutdown. The port goes err-disabled and stays down until manually restored.

    How to read the diagram

    After running, the Root Bridge appears in gold. All four of its ports are designated and forwarding. On each non-root switch, the Root Port (the port leading toward the root via the lowest-cost path) is the path to the root. Blocked links are shown in red dashed — they exist physically but STP has logically disabled them to prevent loops. If the primary path fails, STP unblocks the redundant link.

    Try setting Switch B priority to 4096 while leaving others at default — B becomes root. Try the "Tied priorities → MAC tiebreaker" preset to see the MAC address break the tie (AA:... has the lowest first octet pair, so Switch A wins).
