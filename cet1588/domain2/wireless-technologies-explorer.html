How Wi-Fi shares the air.

Wireless networks aren't just "Ethernet without cables." Every AP shares a slice of radio spectrum with every other AP in earshot — and with microwaves, Bluetooth, and old cordless phones. This artifact teaches you to plan channels properly (1/6/11 in 2.4 GHz), pick the right Wi-Fi generation, configure security (WPA2/WPA3, Personal vs Enterprise), choose antennas for the job, and design coverage that supports roaming.

The complete reference.

Wireless networking is layered: radios occupy frequency bands (2.4/5/6 GHz), divided into channels, transmitting frames according to a standard (Wi-Fi 4/5/6/6E/7), with traffic secured by an authentication and encryption stack (WPA2/WPA3). Access points come in two flavors: standalone (autonomous) or centrally managed (controller-based). Add antenna selection and proper placement for coverage planning.

Frequency Bands — The Big Three
2.4 GHz

Long range, crowded

Range: ~300 ft indoors. Penetrates walls well. Only 3 non-overlapping channels (1, 6, 11) in the US, 13 channels available but they overlap heavily.

CROWDED: Bluetooth, microwave ovens, cordless phones, baby monitors, and every neighbor's Wi-Fi.

CHANNELS: 1-11 (US), 1-13 (EU), 1-14 (Japan, ch 14 only for 802.11b).

5 GHz

Higher throughput, shorter range

Range: ~150 ft indoors. Worse wall penetration. ~24 non-overlapping 20 MHz channels. Much less interference because fewer consumer devices use it.

DFS: Many 5 GHz channels are shared with radar. APs must use Dynamic Frequency Selection — listen for radar, vacate the channel if detected.

WIDTH: Supports 40/80/160 MHz channel widths for higher throughput.

6 GHz · Wi-Fi 6E+

New, uncluttered, fast

Range: similar to 5 GHz, slightly worse. ~59 non-overlapping 20 MHz channels. No legacy devices — only Wi-Fi 6E and Wi-Fi 7 capable clients.

WIDTH: Supports up to 320 MHz channel widths (Wi-Fi 7).

CATCH: Worse wall penetration than 5 GHz. Limited client adoption (only Wi-Fi 6E+).

CHANNEL WIDTH

Wider = faster but greedier

Default is 20 MHz. Wider channels (40/80/160 MHz) provide higher throughput but use more spectrum — fewer non-overlapping channels available.

2.4 GHz: Avoid 40 MHz. Only one 40 MHz channel fits without overlap.

5 GHz: 80 MHz is standard. 160 MHz where supported by clients and free spectrum.

SSIDs, BSSIDs, and ESSIDs
SSID

The network name you see

"Service Set Identifier" — the human-readable network name (e.g., "CBT-WiFi"). Up to 32 characters. Can be hidden (not broadcast in beacons), but that's not a security feature — easily discovered by sniffing.

BSSID

One specific AP's MAC

"Basic Service Set Identifier" — the MAC address of a specific AP radio. One BSSID per AP per band. Clients see this in the air; users see only the SSID.

ESSID

The roaming group

"Extended Service Set Identifier" — multiple APs sharing the same SSID act as one roaming domain. A client moves between BSSIDs (APs) without seeing the network name change.

MULTI-SSID

One AP, many networks

One physical AP can broadcast multiple SSIDs (Corporate, Guest, IoT) each mapped to a different VLAN. Each gets its own BSSID. Common in enterprise to segregate traffic.

Security — WPA Generations
StandardYearEncryptionStatus
WEP1997RC4 (broken)NEVER USE — crackable in minutes

PERSONAL

One shared password

WPA2-Personal uses a pre-shared key (PSK) — everyone uses the same password. WPA3-Personal replaces PSK with SAE (Simultaneous Authentication of Equals), resistant to offline dictionary attacks.

USE: Home networks. Small offices without a directory.

ENTERPRISE

Per-user authentication

WPA2/WPA3-Enterprise uses 802.1X — each user authenticates against a RADIUS server with their own credentials (typically Active Directory). When someone leaves, you disable their account, not change a shared password.

USE: Any organization with a directory. Healthcare, education, business.

PMF

Protected Management Frames

Encrypts management frames (disassociation, deauthentication). Stops deauth attacks that knock clients off Wi-Fi. Mandatory in WPA3, optional in WPA2.

EAP TYPES

How Enterprise auths

EAP-TLS: Cert on both server and client. Most secure. Common in BYOD-strict environments.
PEAP: Server cert only; client password (often MSCHAPv2). Common in Windows shops.
EAP-FAST: Cisco. Replaces older LEAP. Uses a Protected Access Credential.

Antennas
OMNIDIRECTIONAL

360° in the horizontal plane

Default for most APs. Signal radiates outward in a "donut" pattern (no signal directly above or below). Good for general coverage of a room or floor.

USE: Standard indoor AP coverage. Most ceiling-mount APs.

DIRECTIONAL · YAGI

Pencil-beam, medium gain

Focused signal in one direction. Looks like a TV antenna. Used for point-to-point bridges between buildings up to a few miles apart.

USE: Linking two buildings on a campus. Outdoor line-of-sight.

DIRECTIONAL · PARABOLIC

Highest gain, narrow beam

Dish-shaped, focuses signal extremely narrow. Used for long-distance point-to-point links over many miles.

USE: Wireless ISP backhaul. Multi-mile bridges with line of sight.

PATCH / PANEL

Sector coverage

Flat panel that radiates in a defined sector (e.g., 60°-120° forward). Mounted on walls to cover a specific area (warehouse aisle, classroom).

USE: Indoor wall mount with directional coverage. Carrier base station sectors.

AP Architecture — Autonomous vs Controller-Based
AUTONOMOUS

Standalone, configured one at a time

Each AP has its own config — SSID, security, VLANs, channel. No central controller. Manageable up to maybe 5-10 APs.

USE: Small office or single-site home installations.

CONTROLLER-BASED

Lightweight APs + WLC

APs are "thin" — they don't store config. They tunnel back to a Wireless LAN Controller (WLC) using CAPWAP. Controller pushes config, manages roaming, balances load.

USE: Anything with more than ~10 APs. Hospitals, campuses, hotels.

CLOUD-MANAGED

Controller in the cloud

Modern variant — APs report to a cloud dashboard (Cisco Meraki, Ubiquiti UniFi Cloud, Aruba Central). No on-prem controller needed. Easy multi-site management.

USE: Distributed branches, lean IT teams.

CAPWAP

The thin-AP control protocol

"Control and Provisioning of Wireless Access Points" — RFC 5415. Encapsulates AP-to-controller traffic. Replaced older LWAPP. Runs over UDP 5246 (control) and 5247 (data).

MIMO, MU-MIMO, OFDMA
MIMO

Multiple antennas, one client at a time

"Multiple Input Multiple Output" — uses multiple antennas to send/receive multiple data streams simultaneously to ONE client at a time. Introduced in 802.11n. Increases throughput per client.

MU-MIMO

Multiple clients, simultaneously

"Multi-User MIMO" — same time slot, different spatial streams to DIFFERENT clients. Wi-Fi 5 added downlink MU-MIMO. Wi-Fi 6 added uplink MU-MIMO.

OFDMA

Sub-channels per client

"Orthogonal Frequency-Division Multiple Access" — splits a channel into sub-channels (resource units) that can be assigned to different clients simultaneously. Introduced in Wi-Fi 6 (802.11ax). Massive improvement for dense environments with many small clients.

BEAMFORMING

Directional signal shaping

The AP electronically focuses signal toward a specific client instead of radiating equally. Improves range and reliability. Standardized in 802.11ac.

Roaming & Site Surveys
ROAMING BASICS

Same SSID, overlapping coverage

For seamless roaming, all APs must broadcast the same SSID and have overlapping coverage zones (~15-20% overlap). The client decides when to switch to a stronger AP.

FAST TRANSITION

802.11r, k, v

802.11r: Fast BSS Transition — skip 4-way handshake on roaming. Critical for VoIP.
802.11k: Neighbor reports — AP tells the client about nearby APs.
802.11v: BSS transition management — AP can suggest the client should move.

SITE SURVEY

Plan before you deploy

A walk-through measurement of signal strength, interference, and AP placement before deployment. Produces a heat map showing coverage. Tools: Ekahau, NetSpot, AirMagnet.

CAPTIVE PORTAL

The "agree to terms" page

Intercepts new clients and forces them through a web page (terms, login, hotel room number) before they get internet. Common in hotels, airports, guest networks.

Common Exam Traps
  • 2.4 GHz non-overlapping channels are ONLY 1, 6, and 11. Any other combination overlaps. Memorize this.
  • Wi-Fi 5 (802.11ac) is 5 GHz only. 802.11a is also 5 GHz only. Both end in "a" — useful mnemonic.
  • Wi-Fi generation numbers started with 802.11n = Wi-Fi 4. 802.11a/b/g have no generation number.
  • WPA3-Personal uses SAE, not PSK. SAE = Simultaneous Authentication of Equals (Dragonfly handshake). Replaces the vulnerable 4-way PSK handshake.
  • Hidden SSID is NOT security. Easily discovered by sniffing the air. Real security is WPA2/WPA3 + good password.
  • Enterprise = 802.1X + RADIUS. Personal = pre-shared key (or SAE in WPA3).
  • Yagi and parabolic = directional (point-to-point). Omnidirectional is default for AP coverage.
  • CAPWAP = thin AP to controller protocol. Without controller, APs are autonomous and configured individually.

5 GHz Comparison · Why It's Easier

5 GHz has ~24 non-overlapping 20 MHz channels in the US. Even with 80 MHz wide channels (more throughput), you still get ~6 non-overlapping channels — plenty for a dense AP deployment.

How to read this

  • Best practice (1, 6, 11) — three APs each get a clean 22 MHz of spectrum with no overlap. Maximum capacity in 2.4 GHz.
  • Adjacent-channel interference — two APs on nearby (but not same) channels (e.g., 1 and 4) overlap in frequency. Both APs end up wasting airtime fighting through each other's signal.
  • Co-channel interference — two APs on the SAME channel. Slightly less bad than adjacent-channel because the radios coordinate via CSMA/CA (they take turns), but capacity is halved.
  • 5 GHz is just easier. Plenty of channels, less consumer device clutter. Plan with 80 MHz wide channels in mind for most enterprise deployments.
