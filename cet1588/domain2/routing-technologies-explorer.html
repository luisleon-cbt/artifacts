How routers choose paths.

Routing is the Layer 3 decision process: when a packet arrives, the router compares its destination IP against every entry in the routing table and picks the best match. This artifact walks you through the mechanics: longest-prefix-match, administrative distance, metric tiebreakers, the five common protocols (static, RIP, OSPF, EIGRP, BGP), and the FHRP family (HSRP, VRRP, GLBP) that keeps default gateways alive when a router fails.

The complete reference.

Routing is how a router decides where to forward a packet. The router maintains a routing table — a list of known destination networks, each with a next-hop and a cost. When a packet arrives, the router compares the destination IP against the table and applies three tiebreakers in order: longest prefix match → lowest administrative distance → lowest metric.

Static vs Dynamic Routing
STATIC

Manually configured routes

The administrator types each route into the router. No protocol traffic, no CPU overhead, complete control.

PROS: Predictable. Secure (no advertisements). Zero protocol overhead.

CONS: Doesn't adapt to failures. Doesn't scale beyond ~50 routes.

USE WHEN: Stub networks, small office routers, default routes pointing at an ISP.

DYNAMIC

Routes learned from neighbors

Routers run a protocol (OSPF, EIGRP, BGP, RIP) and exchange route information automatically. Adapts to topology changes.

PROS: Self-healing. Scales to thousands of routes. Reacts to link failures.

CONS: Protocol overhead. More complex to troubleshoot.

USE WHEN: Any network with redundant paths or more than a handful of routers.

Default Route — the catch-all

A default route is the route of last resort. Written as 0.0.0.0/0 in IPv4 or ::/0 in IPv6, it matches any destination not covered by a more specific route. Almost every router has one pointing at the next hop toward the internet — typically the ISP. Without a default route, packets to unknown destinations are silently dropped.

Administrative Distance — protocol trust ranking

When multiple routing sources offer a route to the same destination with the same prefix length, the router picks the source with the lowest AD. AD is a measure of trustworthiness — lower means more trusted. Memorize the common values:

VLSM — Variable Length Subnet Masking

VLSM is the practice of subnetting a network into different-sized subnets based on actual host requirements, rather than splitting everything into equal-sized pieces. Saves address space and reflects real-world needs (a 2-host point-to-point link doesn't need 254 addresses).

Example: a /24 (256 addresses) might split into a /25 (126 hosts for the office LAN), a /27 (30 hosts for a small department), several /30 (2 hosts each for router-to-router WAN links). All from the same parent /24, all working together via CIDR-aware routing.

NAT and PAT — Network/Port Address Translation
NAT (1-to-1)

Network Address Translation

Maps one private IP to one public IP. Each internal host gets a dedicated public address. Used when each server needs its own public IP — rare in modern networks.

PAT (1-to-many)

Port Address Translation (NAT overload)

Maps many private IPs to one public IP, using different source ports to distinguish flows. This is what your home router does — every device in your house shares one public IP. The dominant form of NAT today.

Sub-Interfaces — router-on-a-stick

A single physical router interface can be split into multiple logical sub-interfaces, each handling a different VLAN. Notation: GigabitEthernet0/1.10 is sub-interface 10 on physical interface Gi0/1. Each sub-interface gets its own IP and 802.1Q VLAN tag. This is the classic "router on a stick" design for inter-VLAN routing on smaller networks before Layer-3 switches became affordable.

First-Hop Redundancy Protocols (FHRP)

End hosts have one configured default gateway. If that gateway router dies, every host on the subnet loses internet access — until they're manually reconfigured. FHRP solves this: two or more routers share a virtual IP that hosts use as the gateway. If the active router fails, the standby takes over the virtual IP transparently. Hosts don't notice.

HSRP

Hot Standby Router Protocol

Cisco proprietary. One active, one or more standby. Only the active router forwards traffic at any time.

USE: All-Cisco environments.

VRRP

Virtual Router Redundancy Protocol

Open standard (RFC 5798). Functionally similar to HSRP but works across multi-vendor environments.

USE: Mixed-vendor networks (Cisco + Juniper + Arista, etc.).

GLBP

Gateway Load Balancing Protocol

Cisco proprietary. Unlike HSRP/VRRP, GLBP load-balances traffic across multiple routers simultaneously — not just active/standby.

USE: Cisco environments where you want active-active load distribution.

CRP / VRRPv3

Modern alternatives

VRRPv3 supports IPv6. Common Address Redundancy Protocol (CARP) is the open BSD/firewall equivalent.

USE: IPv6 redundancy, open-source firewall HA (pfSense, OPNsense).

Algorithm Families — How Protocols Compute Paths
DISTANCE-VECTOR

"Tell me what you know"

Each router shares its entire routing table with neighbors periodically. Routes are evaluated by hop count or composite metric. Simple but slow to converge.

EXAMPLES: RIP, RIPv2, IGRP (legacy).

LINK-STATE

"Tell me what's connected"

Each router floods its link information to all routers in the area. Every router builds an identical topology map and computes shortest paths using Dijkstra. Fast convergence, scales well.

EXAMPLES: OSPF, IS-IS.

PATH-VECTOR

"Tell me the AS path"

Used between autonomous systems on the internet. Routes carry the full list of AS numbers they've traversed — used for loop prevention and policy.

EXAMPLES: BGP.

HYBRID (Advanced D-V)

"Best of both"

EIGRP technically a distance-vector protocol but uses partial updates and DUAL algorithm for fast loop-free convergence. Behaves more like a link-state protocol.

EXAMPLES: EIGRP (Cisco).

Common Exam Traps
  • "Lowest AD wins, lowest metric wins" — order matters. AD is checked FIRST after longest prefix match. Metric is only consulted when two routes have the same AD (same protocol).
  • BGP comes in two flavors with very different AD values. External BGP (eBGP, between ASes): AD 20. Internal BGP (iBGP, inside one AS): AD 200.
  • HSRP is Cisco; VRRP is open standard. If a question mentions a mixed-vendor environment, the answer is VRRP. If it specifies Cisco-only and load balancing, GLBP.
  • RIP's 15-hop limit. RIP considers a destination unreachable at 16 hops. Why RIP doesn't scale.
  • OSPF requires areas for large deployments. Single-area OSPF (Area 0 only) works for small networks. Large networks divide into multiple areas to limit LSA flooding.
  • "Connected" beats "Static" beats everything else. AD 0 → 1 → dynamic protocols. A static route to a network the router is directly connected to is silently overridden by the connected route.
