Special Address Ranges

Memorize these — every one of them is a fair-game exam question.

RFC 1918 Private — Class A

10.0.0.0/8

16,777,216 addresses. The largest private block. Common in larger enterprise networks because there's no risk of running out.

RANGE10.0.0.0 – 10.255.255.255

RFC 1918 Private — Class B

172.16.0.0/12

1,048,576 addresses. Common in mid-size networks. The /12 means the second octet ranges from 16 to 31.

RANGE172.16.0.0 – 172.31.255.255

RFC 1918 Private — Class C

192.168.0.0/16

65,536 addresses. The default for almost every consumer router (typically subnetted as 192.168.1.0/24).

RANGE192.168.0.0 – 192.168.255.255

APIPA — Auto Private IP

169.254.0.0/16

Self-assigned by Windows/Linux when DHCP fails. If a user reports a 169.254.x.x address, they did not get an IP from DHCP — investigate the DHCP server, cable, or Wi-Fi association.

RANGE169.254.0.0 – 169.254.255.255

Loopback / localhost

127.0.0.0/8

Refers to the local machine itself. Almost always used as 127.0.0.1. Useful for testing whether the local TCP/IP stack is working.

COMMON127.0.0.1 → "localhost"

Limited Broadcast

255.255.255.255

Sent to all hosts on the local network segment. Does NOT cross routers. Used by DHCP DISCOVER and ARP requests.

SCOPELocal broadcast domain only

About this Tool

An interactive Network+ exam prep tool for CompTIA N10-009 Objective 1.7 — IPv4 network addressing. Covers public vs. private (RFC 1918, APIPA, loopback), IPv4 address classes (A through E), and subnetting (VLSM, CIDR).

Built for the calculation-heavy nature of this objective. Use the calculator to verify your work, the visualizer to see the math in binary, and the practice problems to drill the patterns the exam will test.
