Broadcast Domains

Each VLAN is its own broadcast domain. Hosts in VLAN 10 cannot directly reach hosts in VLAN 20 — even though they're on the same physical switch. To cross VLANs you need a router (or Layer 3 switch with inter-VLAN routing).

Access vs Trunk

Access port belongs to one VLAN — frames carry no tag (the switch knows the VLAN by which port the frame arrived on). Trunk port (802.1Q) carries multiple VLANs — frames are tagged with a 4-byte VLAN ID inserted into the Ethernet header.

Spanning Tree Protocol (STP)

Prevents loops in switched networks by selectively blocking ports. Without STP, a single redundant cable causes a broadcast storm that takes down the whole network.

Port States — From Boot to Forwarding

When a switch port comes up with STP enabled, it walks through these states

STP elects a root bridge, calculates the shortest path to it from every switch, and blocks redundant links that would create loops. The states below run on every port at boot and after any topology change.

50 seconds total from cable plug to forwarding traffic in classic STP. RSTP (Rapid Spanning Tree, 802.1w) shortens this dramatically — typically <6 seconds. PortFast / Edge Port skips the full state machine for end-host ports (PCs, printers) since they can't create loops.

Power over Ethernet (PoE)

Delivers electrical power through the same Ethernet cable that carries data. Eliminates separate power runs for APs, IP phones, IP cameras, and other low-power devices.

PoE Standards & Wattage

Each generation increases the power budget per port

The exam expects you to match the standard to its power level — and to know which devices need which level. APs and phones often work with PoE+. Higher-power APs, PTZ cameras, and digital signage need PoE++.

Critical concept: the switch is the PSE (Power Sourcing Equipment), the AP/phone is the PD (Powered Device). The PSE detects whether a connected device is a valid PD before sending power, so plugging a regular PC into a PoE port is safe — the switch won't blast power into a non-PoE device.

MTU & Jumbo Frames

Maximum Transmission Unit — the biggest payload size a frame can carry. Default is 1500 bytes. Jumbo frames push that to 9000.

Why MTU Matters

Bigger frames = less overhead = better throughput for bulk transfers

Every frame has fixed overhead (headers, FCS). With a 1500-byte payload, that overhead is a small percentage of total bytes. With a 9000-byte payload, the overhead percentage drops dramatically — meaning more of the wire's bandwidth is actually carrying user data. Used heavily in storage networks (iSCSI, NFS) and backbone links.

Configuration gotcha: jumbo frames must be enabled end-to-end. If even one switch in the path has the default 1500 MTU, frames will be dropped or fragmented. Misconfigured MTU is a common cause of "intermittent slowness" or "some apps work, others don't" tickets.

Link Aggregation (LACP)

Bundles multiple physical links into one logical link. More bandwidth, plus failover if one link goes down.

How LACP Works

Two switches negotiate an aggregated link via 802.3ad / LACP

LACP (Link Aggregation Control Protocol, IEEE 802.3ad) is the standard for bundling 2 to 8 links between two switches. Vendor terms vary — Cisco calls it "EtherChannel," but the underlying protocol is LACP.

Both ends must agree. If one switch is set to LACP active and the other isn't running LACP at all, the bundle won't form — or worse, you'll create a Layer 2 loop. Misconfigured aggregation is a classic STP-trips-and-fails scenario.

Interface Configuration

Settings the exam expects you to recognize from a config snippet — speed, duplex, native VLAN, voice VLAN.

SettingWhat It DoesCommon Mistake
Speed / Duplex Hard-codes the port speed (10/100/1000) and duplex (half/full). Default is auto-negotiate. Mismatched manual settings on each end of a link → duplex mismatch → late collisions, slow performance, NOT a hard failure
Access Mode Port belongs to a single VLAN. Frames carry no 802.1Q tag. Plugging a trunk-only device (like another switch) into an access port
Trunk Mode Port carries multiple VLANs, tagging each frame with 802.1Q. VLAN list mismatch on each side → some VLANs work, others don't
Native VLAN The one VLAN whose traffic crosses a trunk WITHOUT a tag. Default is VLAN 1. Different native VLAN on each end → security risk + frames go to wrong VLAN. Always match it on both ends.
Voice VLAN Special VLAN ID announced via CDP/LLDP to attached IP phones, automatically tagged. Forgetting to put the voice VLAN in the trunk's allowed list — phones can't communicate
PortFast / Edge Skips STP listening/learning states for end-host ports. Reduces 30-50s connect delay. Enabling on a port that connects to another switch → enables loops
