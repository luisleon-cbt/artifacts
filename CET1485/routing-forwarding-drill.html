Static vs Dynamic

Two ways routes get into a routing table. Static = an admin typed it. Dynamic = a routing protocol learned it.

Static Routing

An administrator manually configures every route. The router does not learn anything on its own — it only knows what you typed.

When to Use

  • Small networks — 2–5 routers, few subnets
  • Stub networks — only one path in or out
  • Default routes — the catch-all "send unknown traffic this way"
  • Security-sensitive — no protocol traffic, no auto-changes

Trade-offs

  • Pro: simple, predictable, no overhead
  • Pro: lowest AD (1) — wins over dynamic protocols
  • Con: doesn't scale — every change is manual
  • Con: doesn't fail over — if the next hop dies, traffic still goes nowhere
Dynamic Routing

A routing protocol (OSPF, EIGRP, RIP, BGP) running on each router shares route information with its neighbors. The routing table updates automatically when the network changes.

When to Use

  • Larger networks — many routers and subnets
  • Networks with redundancy — multiple paths between sites
  • Frequently-changing topologies — links coming up or down
  • Between organizations — BGP for ISP and inter-AS routing

Trade-offs

  • Pro: automatic discovery and failover
  • Pro: scales to thousands of routes
  • Con: protocol overhead (CPU, bandwidth)
  • Con: more attack surface — protocol messages can be spoofed without authentication
★ Default Route — special case worth remembering

A default route is the "gateway of last resort" — written as 0.0.0.0/0. If no other route in the table matches the packet's destination, the router uses this one. Almost every edge router has a default static route pointing at the ISP. In show ip route output, default static routes appear with S*.

AD vs Metric — don't confuse them

AD chooses between protocols. If OSPF and EIGRP both learn 10.0.5.0/24, EIGRP (90) wins over OSPF (110). Metric chooses within a protocol. If OSPF learns two paths to 10.0.5.0/24, the path with the lower OSPF cost (metric) wins. Different protocols use different metrics — RIP counts hops, OSPF uses bandwidth-based cost, EIGRP uses bandwidth + delay.

The Routing Protocols

The five protocol families the exam expects you to recognize and compare.

RIP / RIPv2

AD 120

The simplest dynamic protocol. Counts hops as its metric — every router crossed = 1 hop. Max 15 hops; 16 = unreachable.

Slow to converge, broadcasts entire routing table every 30 seconds. Largely legacy now.

Distance-vector Open standard 15 hops max

OSPF

AD 110

Link-state protocol. Each router has a complete map of the topology and runs Dijkstra's shortest-path algorithm. Metric is cost, derived from bandwidth.

Open standard, scales well, fast convergence. Most common enterprise IGP.

Link-state Open standard Areas (0 = backbone)

EIGRP

AD 90

Cisco proprietary hybrid. Originally Cisco-only, now an open standard. Combines distance-vector and link-state characteristics.

Metric uses bandwidth + delay. Very fast convergence via DUAL algorithm. Common on Cisco-only networks.

Hybrid Cisco origin DUAL convergence

BGP

AD 20 / 200

The protocol that runs the Internet. Routes between autonomous systems (ASes) — your ISP and every other ISP speak BGP to each other.

Path-vector. Doesn't pick fastest — picks based on policy. eBGP between ASes (AD 20), iBGP within (AD 200).

Path-vector Internet backbone Policy-driven

Other Routing Concepts

Topics related to forwarding the exam will also test.

FHRP

Gateway HA

First Hop Redundancy Protocol — keeps the default gateway up if a router fails. Hosts use one virtual IP; multiple routers share it.

Key implementations: HSRP (Cisco), VRRP (open standard), GLBP (Cisco, with load balancing).

HSRP · Cisco VRRP · Standard GLBP · Cisco

Subinterfaces

Router-on-stick

One physical router interface divided into multiple logical sub-interfaces, each with its own IP and VLAN tag. Lets a single router-trunk-link route between many VLANs.

Commonly called "router on a stick" — the trunk to the switch is the stick.

Logical interfaces 802.1Q tagging Inter-VLAN routing

VLSM & Route Summarization

Efficiency

VLSM (Variable Length Subnet Mask) — different subnet sizes within one network. Summarization — many small subnets advertised as one larger one to neighbors.

Cuts routing table size and reduces protocol overhead.

VLSM Supernetting Aggregation

Longest Prefix Match

Selection rule

When multiple routes match a destination, the router picks the one with the longest prefix (most specific). 10.5.0.0/16 beats 10.0.0.0/8 for destination 10.5.1.1.

This rule applies before AD or metric — most specific route always wins.
